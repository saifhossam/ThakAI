============================================================
LEGAL DOCUMENT PROCESSING PIPELINE: DATA FIELD SPECIFICATIONS
============================================================

STAGE 1: PAGE SPLITTING
------------------------------------------------------------
* document_id: A unique identifier (UUID) assigned to the document.
* total_pages: The total count of logical page segments identified in the document.
* strategy_used: The method employed to detect page breaks (e.g., marker_detection).
* page_boundary_type: Indicates if the identified boundaries are "real".
* corresponds_to_pdf_page: Boolean confirming if text segments align with original PDF layout.
* message: The status of the stage processing (e.g., "processed").

STAGE 2: DOCUMENT PROFILING
------------------------------------------------------------
* document_type: The classification of the document (e.g., "Federal Law").
* language: The identified language of the text (e.g., "Arabic").
* structural_vocabulary: List of objects defining the structural units used.
* division_type: The name of the structural unit (e.g., "Chapter").
* level: The hierarchical rank of the division type.
* numbering style: The format of numbering used (e.g., "roman").
* pattern_description: Text description of how the division is identified.
* regex: The regular expression used to find the structural pattern.
* example: A representative snippet of the detected pattern (e.g., "Chapter I").
* estimated_count: The predicted number of these units within the document.
* hierarchy: Ordered list of structural layers (e.g., ["Chapter", "Article", "Clause"]).

STAGE 3: STRUCTURAL EXTRACTION
------------------------------------------------------------
* total_nodes: The total number of hierarchical components extracted.
* chapters_extracted: Specific count of top-level chapters identified.
* top_level_type: The highest structural unit found in the extraction.
* has_preamble: Boolean indicating the presence of an introductory preamble.

STAGE 4: METADATA EXTRACTION
------------------------------------------------------------
* title: The official name or heading of the legal document.
* law_number: The unique reference number assigned to the law.
* jurisdiction: The legal authority or region applicable to the document.
* effective date: The date the document officially becomes active.
* status: The current legal standing (e.g., "in force").
